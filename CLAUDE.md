# CLAUDE.md — lighthouse

このファイルはClaude Codeが起動時に必ず読む仕様書兼ガードレールです。
全ての作業はここに記載されたルールに従って実行してください。

---

## プロジェクト概要

**目的**: 一般的な大学生活・日常生活に関するガイドをHTML形式で公開し、恒久URLで共有する。

**公開先**: GitHub Pages (`/docs`フォルダを配信元とする)

**対象読者**: 大学生全般（特定個人ではない）

**リポジトリ構成**:
```
lighthouse/
├── CLAUDE.md              ← この仕様書（非公開）
├── setup.sh               ← pre-commitフック設置スクリプト（非公開）
└── docs/
    ├── index.html         ← 自動生成される目次ページ
    └── guides/
        ├── dropbox/
        │   └── index.html
        └── （以降追加されるガイド）
```

---

## 初回セットアップ（リポジトリ作成直後に一度だけ実行）

```bash
bash setup.sh
```

このスクリプトはpre-commitフックを`.git/hooks/`に設置します。
必ずリポジトリのルートディレクトリで実行してください。

---

## ガイド作成ワークフロー

新しいガイドを追加する際は以下の順序で作業してください。

### Step 1: ディレクトリ作成

```bash
mkdir -p docs/guides/[ガイド名]/
```

### Step 2: HTMLガイドを生成

後述する「HTMLガイド品質基準」に従って`docs/guides/[ガイド名]/index.html`を生成してください。

### Step 3: 目次ページを自動更新

`docs/index.html`を再生成してください。
`docs/guides/`配下の全ディレクトリを走査し、各ガイドのタイトル・説明・URLを自動的に反映します。
目次のデザインはガイド群と統一されたビジュアルトーンを保ってください。

### Step 4: PIIチェック（必須・省略禁止）

後述する「個人情報チェック」を必ず実施し、問題がないことを確認してください。

### Step 5: commit & push

```bash
git add docs/
git commit -m "add: [ガイド名]ガイド追加"
git push origin main
```

## HTMLガイド品質基準

### 自己完結性（最重要）

生成するHTMLは**完全に自己完結**していなければなりません。

- 外部CDNのCSS・JSは使用禁止（`<link href="https://...">`, `<script src="https://...">` は禁止）
- Google Fontsも禁止。システムフォントスタックまたはBase64埋め込みフォントを使用
- 画像はBase64インライン埋め込みのみ許可
- 1ファイルで完結し、追加ファイルへの依存がないこと

**理由**: CDN障害・オフライン環境・将来のURL変更に対して閲覧を保証するため。

### デザイン基準

以下の基準でデザインしてください。

**タイポグラフィ**:
- 汎用フォント（Arial, Roboto, Inter）は使用禁止
- コンテキストに合った個性的なシステムフォントの組み合わせを選ぶこと
- 日本語コンテンツには日本語フォントスタックを適切に設定すること

**カラーとテーマ**:
- CSS変数（`--color-*`）でカラーパレットを一元管理すること
- ガイドごとに異なるトーンを持たせてよいが、各ガイド内は一貫性を保つこと
- 毎回同じ配色にしないこと（紫グラデーション等の汎用AI的配色は禁止）

**アニメーション**:
- CSS-onlyのアニメーションを積極的に活用すること
- ページロード時のスタガードリビール（`animation-delay`を使った段階的表示）を実装すること
- スクロール連動・ホバーインタラクションを含めること
- アニメーションは過剰にならず、コンテンツの理解を補助する方向で設計すること

**レイアウト**:
- 非対称・グリッドを破るレイアウト等、創造的な空間構成を試みること
- モバイルレスポンシブ必須（`@media`クエリを含めること）
- 十分な余白を確保し、情報密度をコントロールすること

**UIコンポーネント**:
- 階層構造は視覚的に明確に表現すること
- 手順・ステップはインタラクティブに表示すること（クリックで展開等）
- コードブロックが含まれる場合は適切にスタイリングすること

**品質の判断基準**:
生成したHTMLを見た人が「AIが作った汎用的なもの」ではなく「このコンテンツのために設計されたもの」と感じること。

## 個人情報チェック（PIIガード）

### チェック対象（必須項目）

pushの前に、以下の項目が含まれていないかを**意味的に**確認してください。
機械的なパターンマッチングではなく、文脈を読んで判断してください。

| カテゴリ | 具体例 |
|----------|--------|
| 氏名（実名） | 姓名、ニックネームで個人を特定できるもの |
| 学籍番号 | 大学が発行する番号類 |
| メールアドレス | @を含む文字列全般 |
| 電話番号 | ハイフンあり・なし問わず |
| 住所 | 番地レベルの住所 |
| 個人宛て表現 | 「〇〇へ」「〇〇くん」「息子へ」等 |
| SNSアカウント | @ハンドル名等 |

### チェック結果の報告

問題が検出された場合は**pushを実行せず**、以下を報告してください:

```
⚠️ PIIチェック：問題を検出しました
検出箇所: [ファイルパス] [行番号または該当テキスト]
理由: [なぜ問題と判断したか]
対処: [どう修正すべきか]
```

問題がない場合は以下を報告してください:

```
✅ PIIチェック：問題なし
確認対象: [チェックしたファイル一覧]
```

## 目次ページ（docs/index.html）自動生成ルール

### 走査対象

`docs/guides/`配下の各ディレクトリを走査し、`index.html`が存在するものをガイドとして認識します。

### 表示情報の取得

各ガイドの`index.html`から以下を取得して目次に反映してください:

- `<title>` タグ → ガイドのタイトル
- `<meta name="description">` → ガイドの説明文
- `<meta name="date">` → 作成・更新日（形式: `YYYY-MM-DD`）
- `<meta name="category">` → カテゴリ（例: 大学生活, デジタルツール, 生活）

### 目次ページのデザイン要件

- ガイド群と統一されたビジュアルトーンを持つこと
- カード形式でガイドを一覧表示すること
- カテゴリでフィルタリングできるインタラクションを含めること
- 新しいガイドが上部に表示されるよう日付順にソートすること
- 完全自己完結HTML（外部依存なし）


## 禁止事項（まとめ）

- 外部CDN依存HTMLのpush
- PIIチェック未実施のpush
- `docs/`以外へのHTML配置（管理ファイルの誤公開防止）
- 汎用AI的デザイン（紫グラデーション・Inter/Roboto・無個性なレイアウト）
- `setup.sh`未実施での作業開始（pre-commitフックが機能しないため）
